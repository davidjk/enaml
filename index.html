Declarative Syntax

Enaml's declarative syntax is a superset of Python that makes UI code easier to read, write, and debug. The structure of the UI is reflected in the code, making development more intuitive and more fun.

MVC Design

Enaml enforces strict separation of model, view, and controller so reskinning applications down the line is more efficient. This makes prototyping and rapid application development much less costly over the long-run.

Optimized Layout

Enaml's "secret weapon" is a linear solver for widget layout. Instead of using box layout tools, developers define a system of constraints that makes widget layout less fiddly and more dynamic.

Object Management

Enaml uses Atom, an optimized C++ library, to handle changes in object state so elements on the screen are updated accordingly. Atom makes event management fast and easy.

ENAML is a powerful UI framework for creating rich declarative user interfaces. ENAML combines a domain specific declarative language with a constraints-based layout system, making it easier than ever to create complex and dynamic user interfaces. ENAML applications can be deployed on Windows, OS X, and Linux.

about ENAML

Background

ENAML was created with the simple goal of making UI development more productive by making it more intuitive and enjoyable. Existing toolkits often require using verbose tools that make complex UIs difficult to build and manage. Visual interface builders have their place but often don't cut it when truly complex styling or layout is required. If you like Python and don't like XAML files, C++, or Java, Enaml may be for you.

ENAML is inspired in-part by Qt's QML system, but uses desktop widgets for rendering rather than drawing on a 2D canvas. The framework is easily extendable, allowing developers to define their own widgets, override existing widgets, create a new backend toolkit (Enaml currently supports Qt and Wx, though Qt is preferred), or even hook the runtime to apply their own expression dependency behavior. ENAML's syntax is a superset of the Python language.

Currently, ENAML focuses on rich client UIs. Although web interfaces have matured rapidly over the past decade, performant analytical UIs are still the domain of rich client front-ends. Nevertheless, ENAML's architecture can accommodate the creation of a javascript front-end which would allow browser-based UIs. In the meantime, developers can now build rich client UIs much faster. Users often choose to build a working mockup rather than spend time on detailed wireframes because Enaml's productivity is so high.
